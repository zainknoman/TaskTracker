Design Notes

  • workspace_id denormalized on every table — enables fast RLS checks without joins. Every query is workspace-scoped at the row level.
  • subtasks & documents stay JSONB — they're always loaded with the parent task, never queried independently. No benefit to normalizing.
  • task_activity is append-only — RLS denies UPDATE and DELETE. Immutable audit trail.
  • invitations.token is a public lookup — RLS allows SELECT by token for unauthenticated invite acceptance flow (before the user logs in).
  • assignee_id vs ba field — assignee_id links to real auth users; ba stays as a text field for backward compatibility with the existing team model.
  • Realtime enabled on: tasks, projects, milestones, sprints, notifications — not on task_activity (append-only, polled instead).
